## Phase 1B 課程頁與 Stripe Webhook

今日新增課程方案頁與 Stripe Webhook 自動開通通路。

已完成項目：

* 建立課程方案頁：
  * `app/course/page.tsx`
* 建立 Stripe Webhook route：
  * `app/api/webhooks/stripe/route.ts`
* 補上環境變數範例：
  * `STRIPE_PAYMENT_LINK_COURSE`

課程頁設定：

* 主方案：`NT$ 2,400`
* 方案名稱：研究生畢業加速課程 + 6 個月研究報告 AI 指令產生器
* 續約說明：半年後若未畢業，可用 `NT$ 890 / 6 個月` 續約工具權限
* CTA 使用 `STRIPE_PAYMENT_LINK_COURSE`，若尚未設定則顯示付款連結尚未設定

Webhook 實作邏輯：

* 使用 raw body 驗證 `stripe-signature`
* 使用 `STRIPE_WEBHOOK_SECRET`
* 使用 Node `crypto` HMAC SHA-256 驗證 Stripe v1 signature
* 加入 5 分鐘 timestamp tolerance，降低 replay 風險
* 只處理 `checkout.session.completed`
* 使用 `createAdminClient()` 寫入 Supabase，因 webhook 沒有使用者 session
* 付款成功後寫入或更新 `payments`
* 若能用付款 email 找到 `profiles.id`，則寫入 `course_access`
* 同步更新 `profiles.is_paid`、`profiles.paid_at`、`profiles.course_expires_at`
* 以 `payments.stripe_session_id` 做 idempotency，避免 Stripe 重送 webhook 時重複付款紀錄
* 以 `course_access.payment_id` 查詢既有權限，避免 webhook 重送時重複開通

  * `app/login/page.tsx`
* 建立 OAuth callback route：
  * `app/auth/callback/route.ts`
* 建立全站路由保護：
  * `middleware.ts`
* 補上登入成功後的 Dashboard 首頁：
  * `app/dashboard/page.tsx`

實作邏輯：

* 登入頁使用 `supabase.auth.signInWithOAuth`
* provider 使用 `google`
* `redirectTo` 指向 `${NEXT_PUBLIC_SITE_URL}/auth/callback`
* callback route 使用 `supabase.auth.exchangeCodeForSession(code)` 建立正式 Session
* 成功後導向 `/dashboard`
* middleware 保護 `/dashboard/:path*` 與 `/admin/:path*`
* 未登入使用者存取受保護路由時導向 `/login`
* 已登入但非 admin 存取 `/admin/:path*` 時導向 `/dashboard`

注意事項：

* 後續需要在 Supabase Auth Provider 啟用 Google OAuth。
* Google OAuth redirect URL 需要包含正式站與本機 callback URL。
* `/admin` 頁面尚未建立，但 middleware 保護規則已先完成。
